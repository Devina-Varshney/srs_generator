1         Introduction 

1.1       Purpose 

 

The purpose of the Software Requirements Specification is to describe the specific requirements of the project that are to be met by the implementation effort of .  The product perspectives and functions, constraints and assumptions made in the project have also been included. 

 

This document also contains the user characteristics, entity relations, data flow diagrams and state transition diagrams. The specific requirements are clearly mentioned along with performance requirements and design constraints. 

Lastly, the software system attributes which include reliability, availability, scalability, maintainability, and portability have been mentioned.
